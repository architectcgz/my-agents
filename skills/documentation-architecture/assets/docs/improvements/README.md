# 改进项

本目录记录 agent 发现但尚未完全由代码、文档、项目规则、skill 或 agent policy 处理的改进项。

## 状态目录

- `not-impl/`：已接受或合理的改进想法，但尚未实现。
- `implemented/`：已经由代码、文档、policy、skill 或检查处理的改进。
- `agent-recorded/`：已记录到 agent policy 或 skill，但没有其他项目变更。
- `rejected/`：经过考虑后明确拒绝的想法。
- `archived/`：仅为历史保留的过时或已替代条目。

## 规则

- 不要把本目录当成通用任务 backlog。可直接执行的工作应放在 `docs/todo/` 或 `docs/plan/`。
- 每个条目应说明问题、影响范围、建议方向、当前状态和提升目标。
- 条目被其他位置完全处理后，移动或汇总到 `implemented/`；如果 Git 历史已经足够，也可以删除长正文。
